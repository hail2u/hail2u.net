position: absolute on Replaced Element

置換要素を絶対配置した時のleft: autoの扱いがブラウザーと併用するプロパティーによってまちまちな問題について。手前の要素の幅によっても変わるので、折り返しが起こる時の挙動が微妙に違うことも重なる。Chrome 40の挙動が正しいような気がするがわからない。

left無指定(手前の要素が親に収まらない場合)

Chrome 40とMobile Safri 8、Firefox 35では左下に、Internet Explorer 11でボタンが右上に配置される。

left無指定(手前の要素が親に収まる場合)

Chrome 40では左下に、Firefox 35とMobile Safari 8、Internet Explorer 11でボタンが右上に配置される。

left: 0

Chrome 40とMobile Safri 8、Firefox 35では左下に、Internet Explorer 11でボタンが左上に配置される。
