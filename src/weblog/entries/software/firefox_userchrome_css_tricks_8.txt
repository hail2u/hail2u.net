FirefoxのuserChrome.cssネタ #8
<p><a href="http://www.mozilla.org/products/firefox/" title="Firefox - Rediscover the web">Firefox</a>のメイン・メニューのTools以下にあるイマイチ使い道の無いRead MailとかNew Messageというメニュー項目。Optionsを開いたりとかする時に邪魔くせーとか思ってたので、消してみようかと。<a href="http://www.mozilla.org/products/thunderbird/" title="Thunderbird - Reclaim Your Inbox">Thunderbird</a>だとそれなりに活用できそうだけど、インストールすらしたことがないので知らない。Firefox + Thunderbirdなら<a href="http://www.mozilla.org/products/mozilla1.x/" title="Mozilla Suite - The All-in-One Internet Application Suite">Mozila Suite</a>で良いじゃんとか何とか思ったりとかするけど、そういう問題でもないのか。</p>
<h2>Read MailとかNew Messageとかイラネ</h2>
<pre><code class="language-css">menu[label=&quot;Tools&quot;] menuitem[label*=&quot;Read Mail&quot;] {
  display:none !important;
}

menu[label=&quot;Tools&quot;] menuitem[label=&quot;New Message...&quot;] {
  display:none !important;
}

menu[label=&quot;Tools&quot;] menuitem[label=&quot;New Message...&quot;] + menuseparator {
  display:none !important;
}
</code></pre>
<p>普通に消すとセパレーターが残ってしまうので、<a href="http://www.w3.org/TR/REC-CSS2/selector.html#adjacent-selectors" title="Selectors">隣接セレクタ</a>を使って消してみた。普段は滅多に使わないので、なかなか思い出せなかったり。Extension作ってる時も思ったんですが、<code>menuitem</code>には全部<code>id</code>属性振ってくれると楽だな、と。何か事情があるのだろうか。<code>label</code>だとロケール依存だったりするぽいし(en-USしか使ってないから知らないけど)。</p>
