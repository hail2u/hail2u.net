ターミナル画面のマークアップ

<p>Markdown (やその派生)の<code>pre</code>要素になる記法では、問答無用にその中は<code>code</code>要素でマークアップされる。多くの場合はコードの断片なのでそれでいいんだけど、そうじゃないことももちろんある。例えばCLIプログラムでの操作手順を記載する時のターミナルの画面のコピー。</p>

<p>プログラムによる出力は<code>samp</code>要素で、キーボードによる入力は<code>kbd</code>要素になるので、以下のようにマークアップするのがベストに近い。</p>

<pre><code class="language-html">&lt;pre&gt;&lt;samp&gt;$ &lt;kbd&gt;node --help&lt;/kbd&gt;
Usage: node [options] [ -e script | script.js ] [arguments]
       node debug script.js [arguments]

Options:
...
NODE_DISABLE_COLORS    Set to 1 to disable colors in the REPL

Documentation can be found at http://nodejs.org/
&lt;/samp&gt;&lt;/pre&gt;
</code></pre>

<p>シェル・プログラムの出力である全体をまず<code>samp</code>要素でマークアップし、能動的にキー入力する部分は<code>kbd</code>要素でマークアップする。出力のカラーやターミナルのプロンプトなどをそのままに色を付けたい場合は、<code>span</code>要素に適当な<code>class</code>を振ってマークアップすれば良い。</p>

<hr>

<p>過去ログのマークアップを<a href="https://github.com/hail2u/hail2u.net-weblog/commit/01612541f2dbe4899e77157fc7a3e4183c2d547c">こうなるように</a>直してた。ちょくちょく<code>samp</code>要素は使ってたけど、<code>kbd</code>要素は面倒がってまったく使ってなかった……。間違ったマークアップしてたよりはマシだったと思うことにしよう。</p>
