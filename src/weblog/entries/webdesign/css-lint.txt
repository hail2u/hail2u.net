CSS Lint
<p><a href="http://csslint.net/">CSS Lint</a>をしばらく使っていますが、さすがにLintと名乗るだけあって手厳しいですね。効率的でキレイなCSSを書くために自然とHTMLの見直しをすることになる感じが、過渡期の今にはなかなか良いツールな気がします。<a href="http://jigsaw.w3.org/css-validator/">W3C CSS 検証サービス</a>のCSS3対応がいまいち(<code>border-radius: 4px / 8px;</code>が正しい演算子ではありませんとか怒られたりとか)なこともありますし、もっとみんな使うと良いと思います！ストレスたまる気がしますけどね！</p>

<p>このサイトのCSSだと、</p>

<ul>
<li>セレクタでID使うな</li>
<li><code>width</code>や<code>height</code>と<code>padding</code>や<code>border</code>の併用はするな</li>
<li><code>!important</code>は使うな</li>
</ul>

<p>この辺りで引っかかります。「承知でやってるよ！」とか言いたくなりますけど、そこはぐっと我慢。</p>

<p>気になるのはDisallow qualified headingsという、見出し(<code>h1</code>-<code>h6</code>)要素はセレクタの先頭以外に使うなというルールです。HTML5だとそれは無理な話な気が……。それともクラス振れとでも言うのでしょうか？</p>
