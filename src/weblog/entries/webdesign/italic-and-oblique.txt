<h1>italicとoblique</h1>

<figure class="fill">
<a href="/images/blog/italic-and-oblique.png"><img alt="Georgia Proでのイタリック体と擬似斜体の比較" src="/images/blog/italic-and-oblique.png"></a>
</figure>

<p>イタリック体（シュッと斜めなやつ）とオブリーク体（クタッと斜めなやつ）が違うことは知っていたが、<code>font-style</code>プロパティーの値としての<code>italic</code>と<code>oblique</code>は互換だと勘違いしていた。どちらを指定してもイタリック体かオブリーク体のどちらか存在する方を使ってくれ、両方ない場合は擬似斜体を生成してくれると思っていたが、違うようだ。</p>

<figure>
<blockquote>
<p>A value of &apos;normal&apos; selects a font that is classified as &apos;normal&apos; in the UA&apos;s font database, while &apos;oblique&apos; selects a font that is labeled &apos;oblique&apos;. A value of &apos;italic&apos; selects a font that is labeled &apos;italic&apos;, or, if that is not available, one labeled &apos;oblique&apos;.</p>

<p>The font that is labeled &apos;oblique&apos; in the UA&apos;s font database may actually have been generated by electronically slanting a normal font.</p>
</blockquote>
<figcaption><cite><a href="https://www.w3.org/TR/CSS2/fonts.html#font-styling">15.4 Font styling: the <code>&apos;font-style&apos;</code> property</a></cite></figcaption>
</figure>

<p>仕様の<code>font-style</code>プロパティーの項ではこう書かれている（CSS 2.2でも変わっていない）。つまり<code>normal</code>なら通常書体だけを、<code>oblique</code>ならオブリーク体だけを探すものの、<code>italic</code>ならまずイタリック体を探すが、見つからなかった場合はオブリーク体を探すというわけだ。そしてオブリーク体は場合によっては通常のフォントから機械的に斜めにしたもの（いわゆる擬似斜体）であるかもしれない、というわけだ。</p>

<p>このような対応関係になる。つまりオブリーク体を<code>italic</code>で参照することはできるが、逆にイタリック体を<code>oblique</code>で参照することはできない。</p>

<hr>

<p>というわけでイタリック体があるのに斜体にすらなっていないメイリオの日本語部分で擬似斜体を出す方法があることがわかった。</p>

<p>Demo: <a href="/pub/test/652.html">Faux Oblique for Meiryo</a></p>

<pre><code class="language-css">.test {
  font-family: Meiryo, sans-serif;
  font-style: oblique;
}
</code></pre>

<p>メイリオがインストールされていない環境で、イタリック体があっても強制的に擬似斜体になるという問題がある。そのため使いどころはあまりない。</p>
