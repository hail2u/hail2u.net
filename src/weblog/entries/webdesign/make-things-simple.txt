<h2>ロゴ</h2>

<p>手持ちのフォントで&quot;h&quot;だけを打ち出して良さそうなのを選んだだけという。使ったフォントは<a href="http://new.myfonts.com/fonts/larabie/jandles/">Jandles</a>で、MyFontsから無料で手に入る。ベベルるかグラデるか迷った末グラデった。もうアルファチャンネル付きのPNGファイルでいいよね？いいよね！</p>

<p>ロゴの配置はちょっと悩んだ。マージンに負の値を入れて<code>float</code>させればいけそうな気がしたが、それではコンテンツとかぶさるように配置するのは無理な感じだった。また、コンテンツの左右のマージンが一定ではないので、<code>absolute</code>でも正確に配置できない。とりあえず親要素を<code>relative</code>、ロゴ部分を<code>absolute</code>にしてやって、親要素を基準にしてロゴを絶対配置してみた。</p>

<pre><code class="language-css">#logo {
  position: relative;
}

#logo a {
  <code>margin-right: 12px;</code>
  display: block;
  <em>position: absolute;</em>
  top: 0px;
  <em>right: 100%;</em>
}
</code></pre>

<p>うまくいってるような気がする。<code>left</code>に負の値を指定するよりも、<code>right</code>と<code>margin-right</code>で配置しておくとロゴ(の大きさ)を変えた時にもそのままで良いので少し楽。</p>

<h2>ナビゲーション</h2>

<figure><a href="/pub/test/136.html"><img src="/images/blog/smiley-border.png" alt="顔文字っぽく"></a></figure>

<p>本当はTwitterやフィードのアイコンと合わせたアイコンを作って並べてやろうと考えていたんだけど、作ったはいいものの何のページかわからないというアイコンじゃない落書きになったのでお蔵入り。先にロゴが出来、それが非常にシンプルなものになったので、特に装飾もせずに単なるリンクにした。</p>

<p>Chrome等ではリンクの下の<code>border</code>がAmazonのロゴのようなゆるい曲線になる。特に小難しい事をしているわけではなく、<code>-webkit-border-radius</code>で<code>50%</code>(以上)を指定するだけ。</p>

<pre><code class="language-css">a {
  border-bottom: 2px solid rgb(102, 170, 17);
  -webkit-border-radius: 50%;
}
</code></pre>

<p>これだけでリンクの下に曲線が引ける(ある程度<code>padding-bottom</code>が必要)。<code>border-radius</code>でパーセンテージ指定とか初めて使った。ボックスの幅と高さによって曲線のきつさが変化するので、明示的に曲線のきつさを表現することはできない。FirefoxやOperaではちょっと変な感じになるので、<code>-webkit-border-radius</code>のみにした。</p>

<hr>

<p>どんな英単語も下にゆるい曲線つけるだけで顔文字に見えてくる。不思議！</p>

<h2>追記</h2>

<p>ナビゲーションのリンクの下に出るゆるい曲線を<a href="http://web.g.hatena.ne.jp/vantguarde/20101229/1293604827">) - vantguarde - web:g</a>で教えてもらった左右の<code>border</code>を<code>transparent</code>にする方法でFx3.5以降にも対応させてみた。IE9もいけるらしい。</p>

<pre><code class="language-css">a {
  border: 2px solid transparent;
  border-bottom-color: rgb(102, 170, 17);
  -moz-border-radius: 50% / 12px;
  -webkit-border-radius: 50%;
}
</code></pre>

<p>Fx3.5では%によるサイズ指定がボックス幅基準になってしまい縦方向のサイズ指定がおかしくなるので、垂直方向のサイズをpx指定した(ここのナビゲーションは高さは24px固定なのでその半分の12pxにした)。Fx4では完全対応したらしい。</p>

<p>Op11(たぶん10も)は<code>border-radius</code>での%指定の解釈がCh9やFx4と大きな差があり、バグっぽい挙動でどうしてもうまくいかない。px指定なら綺麗に円になるのにな。直せとは言わないが、せめてベンダー拡張に降格させて欲しい。</p>
