Yahoo! UI LibraryのFonts CSS

<p>OSのみならずブラウザにも左右されるWebにおけるフォント。多くの場合はHelvetica(WindowsではArialになる)やVerdanaなどの一般的なフォントを指定し、多少サイズをいじる程度のアプローチか、まったく無指定でユーザーに任せるといったアプローチであることと思う。徹底的に調節するとなるとものすごくツライから。<a href="http://developer.yahoo.com/yui/" title="Yahoo! UI Library">Yahoo! UI Library</a>で提供されている<a href="http://developer.yahoo.com/yui/fonts/" title="Yahoo! UI Library: Fonts CSS">Fonts CSS</a>はそのフォント調節の手助けをしてくれるCSSファイル。</p>

<p>具体的な効用は、ほとんどの現行ブラウザで13ピクセルのHelveticaが基本のフォントになるというもの。つまり、OSとブラウザによる文字表示の差異をほとんど吸収してくれるという効用。</p>

<p>当該のCSSファイルを開いてみると、微妙に読みづらいCSSコード。よく読んでやると<code>*property: value;</code>というハックのみの模様。このプロパティ名の直前にアスタリスクを置くハックは寡聞にしてどこが初出かわからなかったが、ちょっと試してみたところInternet Explorer 5/6のみがすり抜けるハックっぽい。つまりInternet Explorer以外では<code>13px</code>、Internet Explorerでは<code>small</code>が適用されるということな模様なので、いきなり<code>13px</code>とか指定してるものの「Internet Explorerなんかではフォント・サイズが固定されそう！」とかいう心配は必要ない。</p>

<p>フォント・サイズが13ピクセルなのは英語では良いかなという感じだが、日本語には少しつらいと感じる人も多いかも。そこは自分で書くCSSの方で調節してやれば良い。<code>em</code>ではなく<code>%</code>でサイズの調整はしましょうとのことだ。Fonts CSSのページには16ピクセルにするには何%を指定すれば良いかの表とかもあるので、簡単に調節できるはず。Fonts CSSそのものを編集して調節ということもできそうかなと思うかもしれないが、Internet Explorerの<code>small</code>に合わせて他のブラウザでのサイズを調節している(と思われる)ので、ちょっと難しいと思う。</p>

<p>なんにせよ基準がきっちりと決まるので、CSSを書くのにかかる手間がかなり省けることと思う。Yahoo! UI Libraryで併せて提供されている<a href="http://developer.yahoo.com/yui/reset/" title="Yahoo! UI Library: Reset CSS">Reset CSS</a>も使うと良い。僕は<a href="/styles/common.css" title="common.css">自分で書いたのがある</a>ので使っちゃうけど。</p>
