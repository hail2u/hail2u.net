続・子セレクタ
<p>以前、<a href="/blog/webdesign/css_tips_1.html" title="hail2u.net - Weblog - 子セレクタ">Internet Explorerは子セレクタというCSSのセレクタの文法に対応していないとか書いた</a>んですが、<a href="http://www.web-graphics.com/" title="webgraphics">Web-Graphics</a>の<a href="http://web-graphics.com/mtarchive/001142.php" title="wg:Limited child selector support in IE5.0/Win?">Limited child selector support in IE5.0/Win?</a>によるとInternet Explorer 5.0に限っては部分的に対応しているみたいです。どういう場合かというと、</p>
<pre><code class="language-css">/* ignored */
body &gt; div.test { margin:1em; }

/* not ignored */
#foo &gt; #bar { margin:1em; }
</code></pre>
<p>というような<a href="http://www.w3.org/TR/REC-CSS2/selector.html#id-selectors" title="Cascading Style Sheets, Level 2 - 5 Selectors - 5.9 ID selectors">IDセレクタ</a>と<a href="http://www.w3.org/TR/REC-CSS2/selector.html#child-selectors" title="Cascading Style Sheets, Level 2 - 5 Selectors - 5.6 Child selectors">子セレクタ</a>の組み合わせな場合です。<code>id</code>を割り振った要素に対して子セレクタを使ってCSSを適用するということはまず無いと思うので、あまり影響は無さそうです。</p>
<p>とかいって部分的に対応しているのかと思ったら、Internet Explorer 5.0は&quot;#&quot;以前の文字を無条件に無視するというバグに起因するもののようです。こっちの方が知識として重要ですね。</p>
