Life with RSS開発 #3
<p>なぜか異様にやる気が出てきた。このやる気があるうちにある程度まで作り上げないと放置するな、多分。追加した機能は以下の通り。</p>
<ul>
<li>アイテムの追加を実装</li>
<li>ディレクトリの追加を実装</li>
<li>アイテム・ディレクトリの削除を実装</li>
<li>OPMLファイル保存時に書き込む時刻をWin32 APIでUTCを取得してからフォーマット変換するようにした</li>
<li><a href="http://rss.bulknews.net/" title="Bulkfeeds: top - [rss.bulknews.net]">Bulkfeeds</a>検索時にツリービューに処理中メッセージを表示するようにした</li>
</ul>
<p>アイテム・ディレクトリの移動はまだまるで手をつけてなかったりする。これ完成する頃にはOPML編集ウェブ・サービスとか出来てそうな気がするけど気にしない。てか、それ以前にOPMLはやめて<a href="http://www.w3c.org/RDF/" title="Resource Description Framework (RDF) / W3C Semantic Web Activity">RDF</a>とか<a href="http://internetalchemy.org/ocs/" title="Open Content Syndication">OCS</a>、<a href="http://www.intertwingly.net/wiki/pie/FrontPage" title="FrontPage - Atom Wiki">Atom</a>使ってやろうよとかいう気運があるのも気にしない。</p>
<p>アイテム追加時に元となるURL(Source URL)からRSS Auto-discoveryとか実装したいところだけど、どう実装するかがなかなか難しい。ファジーな探し方をすると負担が大きいので、BulkfeedsでのURL検索から持ってきたりとかしたいです。とか、言ってみる(散々言ったのに)。<a href="http://www.syndic8.com/" title="Syndic8.com - Welcome!">Syndic8.com</a>でもいいのだけれども、日本は弱いし、それ以上にゴミ多い&amp;重過ぎ。やはりここは！ま、ファジーな探し方にこだわらなければ、単にRSS Auto-discoveryを実装してみれば良いだけなのですが。ここらへんはまだまだ後回し。</p>
<p>そろそろ主要RSSリーダーの出力するOPMLファイルをチェックしないとまずそうだ。今のところ確認してるのは<a href="http://www.bloglines.com/" title="Bloglines | Free, Web-Based News Aggregator">Bloglines</a>だけだし。</p>
<p>ていうか、いまいち存在意義がわからないアプリケーションだと思います。大雑把に目指しているものを説明するとすると、</p>
<blockquote>
<p>RSS購読状況のメンテナンスをするもの</p>
</blockquote>
<p>で、ブラウザのブックマークエディタのRSSリーダー用みたいなものだと思ってくれると一番近いんじゃないかなと思います。複数のRSSリーダー間で行動状況を同期できたりメンテナンスできたり。そんなん。</p>
