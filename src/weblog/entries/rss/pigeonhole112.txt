Pigeonhole 1.1.2
<p>Pigeonhole 1.1.2を公開しました。<a href="http://bulkfeeds.net/" title="Bulkfeeds: Home - RSS Directory &amp; Search">Bulkfeeds</a>が、一部機能を利用するのにAPI Keyを必要とすることになったことを受けてのバージョン・アップです。ついでにRSSの存在確認を右クリックから行えるようにしてみたり。</p>
<p>Bulkfeedsのリリースによると、<a href="http://blog.bulknews.net/mt/archives/000888.html" title="blog.bulknews.net: Bulkfeeds API Key 取得を開始">今のところAPI Keyなしのアクセスに制限をつけてはいない</a>ようなので、どういう風に実装するか迷ったのですが、Bulkfeedsを利用する機能(RSSをURLから探す)を利用しようとした時に、API Keyが必要とのダイアログを出し、API Keyを設定するページをアクティブにしてオプション・ダイアログを開いてやるようにしました。「アプリケーションでAPI Keyを取得して、ユーザーは共通で良いですよ」ということも言われたのですが、やはり各自で取得すべきな気がするので必須にしてしまいました。</p>
<figure><img src="/images/blog/pigeonhole_bulkfeeds_api_key_setting.png" alt="PigeonholeのBulkfeeds API Keyの設定"></figure>
<p>まーどうせ誰も使ってねーよ。僕は使ってますけどね。</p>
<p>RSSのステータスチェックは、今のところ非常にショボイものですが、全てチェックして、存在しないものは一括で削除とか出来るようにしようかなとか妄想してたりするかもしれない段階で、そんなことが出来たら良い感じな気がする。一括チェックまでは簡単なんですけどね。その先がメンドイ。</p>
