UglifyJSのHTTP APIを叩くPerlスクリプト

<p><a href="http://marijnhaverbeke.nl/uglifyjs">UglifyJS</a>に<a href="http://code.google.com/p/closure-compiler/">Closure Compiler</a>から乗り換えようかなーとちょっと思っているのでとりあえずコンパイルするためのスクリプトを書くところから。UglifyJSのHTTP APIがそこそこClosure Compiler ServiceのREST APIと互換性があるようなので、まずは<a href="/blog/coding/compile-local-js-file-with-closure-compiler-service.html">ローカルのJavaScriptファイルをClosure Compiler ServiceのREST APIでコンパイルできるように</a>で書いたPerlスクリプトを修正して使ってみる。</p>

<p class="seealso">Download: <a href="/pub/uglifyjs.pl.txt">uglifyjs.pl</a></p>

<p>使い方は元になったClosure Compiler ServiceのREST APIを叩く奴と一緒。</p>

<p>UglifyJSのHTTP APIでは<code>js_code</code>パラメーターが複数使えない。なので手元でまとめてから投げたりする必要があるので、ちゃんとまとめないと意図した順番でコードが実行されない可能性がある。素直にどこかにアップロードしてから<code>code_url</code>パラメーターでそれを指定したほうが安全そう。</p>

<p>Closure Compiler Serviceのような埋め込みコメントに対応したCUIラッパー書いた方が良さそう。ありそう。</p>

<h2>追記</h2>

<p>Closure Compilerに比べてローカル環境を構築するコストと実行するコストが低い気がするので<a href="/pub/concatjs.pl.txt">連結するだけのスクリプト</a>を書いてパイプでつなげることにした。Closure Compilerの<code>@code_url</code>とそれに似た<code>@code_path</code>だけ使えて、それらを連結するだけ。</p>
