SubscriptionIdからAWSAccessKeyIdに
<p><a href="http://www.amazon.com/gp/browse.html?node=12738641" title="Amazon.com Help: E-Commerce Service 4.0">Amazon E-Commerce Service</a>の2005-10-05版より、<code>SubscriptionId</code>パラメータではなく<code>AWSAccessKeyId</code>パラメータというのが推奨になったようです。これまで通り<code>SubscriptionId</code>パラメータも利用できるようですが、新たに<a href="http://aws.amazon.com/" title="Amazon.com Help: Web Services">Amazon Web Services</a>に登録するユーザーにはAccess Key IDしか発行されないので、<code>AWSAccessKeyId</code>パラメータを使ってコーディングするようにした方が良さそう。非公開なアプリケーションなら関係ないですけど。</p>
<p>Access Key IDは<a href="http://aws.amazon.com/" title="Amazon.com Help: Web Services">Amazon Web Services</a>からログインし、ページ右上のYour Web Services Accountにマウスカーソルを合わせると出てくるメニューからView Access Key IdentifiersをクリックするとAccess Key IDが表示されます。</p>
<p><code>SubscriptionId</code>パラメータと<code>AWSAccessKeyId</code>パラメータは特に違いはなさそうな感じなので、入れ替えるだけでOKっぽい。XSLTサービスを利用している場合は<code>Version</code>パラメータに<code>2005-10-05</code>を指定した上で、XSLファイルのnamespaceも修正しなければならなかったりとかちょっと大変かも。</p>
