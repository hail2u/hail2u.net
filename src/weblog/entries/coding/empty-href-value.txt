空文字列のリンク

<p><code>a</code>要素の<code>href</code>属性で空文字列を指定した時の挙動はその文書の再読み込みになると思っていた。けど、どうもInternet Explorer 10 (やもっと古いバージョン)では<code>./</code>と同じになるようだ。確実にそのものを再読み込みするためには明示的にその文書自身のURLを指定するしかなさそう。</p>

<p class="seealso">View Demo: <a href="/pub/test/500.html">Empty href Value</a></p>

<p>Chrome 29やFirefox 24では再読み込みになる。この挙動が便利でたまにどうでも良いページで使っていたけど、Internet Explorer 10では<code>./</code>になりディレクトリの参照とかになるのは知らなかった。空文字列だった場合のURL解決方法は<a href="http://url.spec.whatwg.org/#relative-state">relative state</a>でいきなりEOF code pointが来たケースだと思うので、Internet Explorer 10のバグのような感じだ。</p>

<p>URL Standardsに従うと<a href="/pub/test/500-2.html"><code>base</code>要素と組み合わせた場合</a>は再読み込みにはならないとかもあるので、僕みたいに空文字列で再読み込みみたいな斬新な覚え方してるとハマる。</p>
