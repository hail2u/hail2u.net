Amazon ECS 4.0のXMLからフィードやJSONに
<p><a href="http://www.amazon.com/gp/browse.html?node=3435361" title="Amazon.com Help:">Amazon Web Services</a>のE-Commerce Service 4.0ではクエリに<code>Style</code>というパラメータでXSLTスタイルシートのURLを指定することによって返ってくる結果のXMLを整形することが可能です。前からありますけど。久々にAmazon熱が出たので、RSS 2.0とJSONに整形するXSLTスタイルシートを書いてみました。</p>
<ul>
<li><a href="http://webservices.amazon.co.jp/onca/xml?Service=AWSECommerceService&amp;SubscriptionId=1T2SAFDMWVQS3E47MD02&amp;AssociateTag=hail2unet-22&amp;ResponseGroup=Medium&amp;Operation=ItemSearch&amp;SearchIndex=Books&amp;Keywords=Perl&amp;Sort=daterank&amp;Version=2005-07-26&amp;Style=http://hail2u.net/pub/ecs42rss20.xsl" title="Amazon.co.jpの和書ストアから &quot;Perl&quot; を検索した結果をRSS 2.0に整形">RSS 2.0で整形</a> (XSLTスタイルシート: <a href="/pub/ecs42rss20.xsl" title="ecs42rss20.xsl">ecs42rss20.xsl</a>)</li>
<li><a href="http://webservices.amazon.co.jp/onca/xml?Service=AWSECommerceService&amp;SubscriptionId=1T2SAFDMWVQS3E47MD02&amp;AssociateTag=hail2unet-22&amp;ResponseGroup=Medium&amp;Operation=ItemSearch&amp;SearchIndex=Books&amp;Keywords=Perl&amp;Sort=daterank&amp;Version=2005-07-26&amp;ContentType=text/javascript&amp;Style=http://hail2u.net/pub/ecs42json.xsl" title="Amazon.co.jpの和書ストアから &quot;Perl&quot; を検索した結果をJSONに整形">JSONで整形</a> (XSLTスタイルシート: <a href="/pub/ecs42json.xsl" title="ecs42json.xsl">ecs42json.xsl</a>)</li>
</ul>
<p>JSONで出力する場合は、クエリに<code>Style</code>と共に<code>ContentType</code>というパラメータで<code>text/javascript</code>を指定してやると良さげです。あーあと<a href="http://la.ma.la/blog/" title="最速インターフェース研究会">最速の人</a>に教えてもらって、Internet Explorer対策に<code>null</code>とか仕込んでます。こうしないとカンマで終わってるところでエラーが出ちゃうので。</p>
<p>XSLTスタイルシートは両者共に超決め打ちなので、このままでは使い物になりませんが、叩き台程度にはなるんじゃないかなーと。RSS 2.0の方はそのまんまRSSリーダーにぶち込んで良いですし、JSONの方はサーバーのリソースを消費しないAmazon検索システムとかで。最速の人の<a href="http://la.ma.la/misc/aws/" title="Amazon最速検索 説明書">Amazon最速検索</a>はそういうアプローチのものですね。JSONで出力すると、FirefoxのExtensionなんかでも扱いやすいかも。</p>
