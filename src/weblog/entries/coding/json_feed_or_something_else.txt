JSON Feedとか何とか
<p><a href="http://d.hatena.ne.jp/secondlife/" title="川o・-・）＜2nd life">僕をADT呼ばわりする二人の片割れの方</a>が<a href="http://d.hatena.ne.jp/secondlife/20050825/p1" title="川o・-・）＜2nd life - JSON Feed">JSON Feedネタをぶちあげた</a>ので、「先に書かれた！」とか思いつつ対抗して書いてみる。ADTなので！てなわけでblosxomのflavourで<a href="/pub/index.json">作ってみました</a>(今後ちゃんと更新されるかどうかは未定です)。このJSON Feedのサンプルは、<a href="/pub/json_flavour.zip" title="/pub/json_flavour.zip">アーカイブにまとめたflavour</a>のみで生成しています。使い方とかは2nd lifeの当該エントリに書いてあるので、省略。</p>
<p>JSON Feedとか銘打って勝手なことをやってますが、これはHTML+CSS+JavaScriptなリッチ・クライアントにはもってこいな形でフィード、つまり<a href="http://blogs.law.harvard.edu/tech/rss" title="RSS 2.0 Specification">RSS 2.0</a>や<a href="http://www.atomenabled.org/developers/syndication/atom-format-spec.php" title="AtomEnabled / Developers / Syndication / Atom Syndication Format Spec (Pre-Draft)">Atom 1.0</a>と似たようなものを提供してやると幸せな人がいるかもいないかもとかいう妄想の類の話です。作成したJSON Feedにコメントでなんやかんや書き込んでありますが、何の根拠も無いです。</p>
<p>形的には<code>jsonfeeds</code>というシロモノに、そのJSON FeedのURLを添え字としてRSSやAtom相当の情報をぶち込むという形です。単純に<code>jsonfeed</code>とかいう何かにぶち込まなかったのは、複数のJSON Feedをダーッと読み込んで、後でまとめて処理するとかすることがやりづらくなっちゃいそうだからです(出来ないことはない)。URLが添え字で長いですが、実際に扱うときはあんまり意識することはないんじゃないかなとか。</p>
<p>それぞれのプロパティ名は適当。RSS 2.0と似たような感じ。<code>description</code>とかも作ってもいいかも。<code>date</code>とか超適当。更新日時か公開日時かわかんねぇよ。日付の形式を<a href="http://rfc.net/rfc822.html" title="RFC822">RFC822</a>に従ったものにしたのは、JavaScriptにそのまんま投げられそうとか、<a href="http://www.w3.org/TR/NOTE-datetime" title="Date and Time Formats">W3CDTF</a>は<a href="http://www.blosxom.com/" title="blosxom :: the zen of blogging ::">blosxom</a>ではちょっと苦労するとかそんなダメな理由です。Epoch Timeとかでもよさげな気がする。</p>
<p>めざせIETF！</p>
