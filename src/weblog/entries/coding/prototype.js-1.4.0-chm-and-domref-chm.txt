prototype.jsの開発メモとGecko DOM ReferenceのHTMLヘルプ

<p>未だにちょくちょくみてしまう<a href="http://prototype.conio.net/">prototype.js</a>の簡単なリファレンスである<a href="http://www.sergiopereira.com/articles/prototype.js.html">Developer Notes for prototype.js</a>の<a href="http://www.imgsrc.co.jp/%7Ekuriyama/prototype/prototype.js.html">日本語版</a>と<a href="http://www.mozilla.org/docs/dom/">The Document Object Model in Mozilla</a>で配布されているGecko DOM ReferenceのZIP version(古い奴)をHTMLヘルプにしてみた。HTMLヘルプはちょっと作ってみちゃったりすると、HDD上にあるありとあらゆるHTMLのドキュメントをHTMLヘルプにしたい欲求に駆られてしまう・・・のは僕だけですか、そうですか。</p>

<ul>
<li><a href="/archives/files/prototype.js-1.4.0.zip">prototype.jsの開発メモ</a></li>
<li><a href="/archives/files/domref.zip">Gecko DOM Reference</a></li>
</ul>

<p>prototype.jsの開発メモの方はHTMLに手を入れて、メソッドやプロパティに<code>id</code>を振り、キーワードでジャンプできるようにした。HTMLヘルプ向けにCSSもちょっと調整。チェックしきれてない説があるので、「このキーワードが変」とかあったらメールとかブクマ・コメントとかで。キーワードの割り当てが微妙な気がする。</p>

<p>実際には<a href="http://script.aculo.us/">script.aculo.us</a>に同梱されている1.5.0_rc1やSubversionリポジトリから拾ってきた最近のものを使っている人がほとんどだと思うので、使い道があまり無い。これからprototype.js触ってみるとかいう人には良いかも。</p>

<p>Gecko DOM Referenceの方はToCとIndexがあるのでそれらからほぼ自動生成。</p>

<p>HTMLヘルプは元になるHTMLさえあれば結構簡単に作れるので、いろいろと自分で作ってみてはどうでしょうか。そして作ったら公開してほしいなぁ、とか思いました。まる。コペ。コペ。そろそろ作り飽きてきた。</p>
