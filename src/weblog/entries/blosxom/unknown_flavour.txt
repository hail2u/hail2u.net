不明なflavour
<p><a href="http://www.blosxom.com/" title="blosxom :: the zen of blogging ::">blosxom</a>は頑張りやさんなので、予め用意していないflavourを訪問者が指定してアクセスしてきた場合もどうにか頑張ってくれちゃったりします(わけのわからないフリ)。その時はerrorというblosxom.cgi内で定義されたflavourが使用されます。そのerror flavourの替わりに、<code>$default_flavour</code>を利用するようにする改変方法が<a href="http://agilitynerd.com/index.cgi/tech/web/blogging/blosxom/UnknownFlavourError.html" title="AgilityNerd">AgilityNerdで公開されていました</a>。</p>
<p>とりあえずうちのblosxomにも適用してみたところ、ちゃんと動いているようです。とは言うもののほぼ問答無用で<code>$default_flavour</code>な感じなので、思っても見ないところで不具合が出そうな気がしたりも。</p>
<p>このAgilityNerdの不明なflavourへの対策は、<a href="http://blosxom.ookee.com/blog/" title="The Unofficial Blosxom User Group">The Unofficial Blosxom User Group</a>経由で知りました。The Unofficial Blosxom User Groupは世界で唯一と言って良いアクティブなblosxomの情報サイトです。MLでのプラグイン公開のアナウンスなどを即座に投稿してたりとなかなか精力的で、blosxomユーザーには得るところが多いサイトだと思います。</p>
