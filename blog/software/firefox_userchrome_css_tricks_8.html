FirefoxのuserChrome.cssネタ #8

Dec 10, 2004 · Software

Firefoxのメイン・メニューのTools以下にあるイマイチ使い道の無いRead MailとかNew Messageというメニュー項目。Optionsを開いたりとかする時に邪魔くせーとか思ってたので、消してみようかと。Thunderbirdだとそれなりに活用できそうだけど、インストールすらしたことがないので知らない。Firefox + ThunderbirdならMozila Suiteで良いじゃんとか何とか思ったりとかするけど、そういう問題でもないのか。

Read MailとかNew Messageとかイラネ

menu[label="Tools"] menuitem[label*="Read Mail"] {
  display:none !important;
}

menu[label="Tools"] menuitem[label="New Message..."] {
  display:none !important;
}

menu[label="Tools"] menuitem[label="New Message..."] + menuseparator {
  display:none !important;
}

普通に消すとセパレーターが残ってしまうので、隣接セレクタを使って消してみた。普段は滅多に使わないので、なかなか思い出せなかったり。Extension作ってる時も思ったんですが、menuitemには全部id属性振ってくれると楽だな、と。何か事情があるのだろうか。labelだとロケール依存だったりするぽいし(en-USしか使ってないから知らないけど)。

Weblogの一ページ目をFirefoxネタで埋め尽くす計画が何気に進行中。もうちょっとだな。
