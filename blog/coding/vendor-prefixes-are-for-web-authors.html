2013年11月21日

ベンダー拡張プリフィックスはウェブ制作者たちのためにある

ブラウザー開発者はベンダー拡張プリフィックスをなくす方向に動いているような気がする。短いリリース・サイクルの中で、仕様の要件を満たしているのなら外すといった地味な確認作業から開放されたいのかもしれない。そういう理由はわからなくもないが、ウェブ制作者も面倒くさくなくなるのでそれで良い、と考えていそうなことにはまったく賛成出来ない。

CSSグラデーションの実装と仕様の紆余曲折は、ウェブ制作者にベンダー拡張プリフィックスに対する不信感を植え付け、いい加減に付加する口実を与えてしまった。独自に実装、仕様が追随、対立、仕様の変化、対応、どの段階においてもベンダー拡張プリフィックスはさほど効果は上げなかった。結果、書くのがただただ面倒なだけということに。

変化についていくことを諦めたウェブ制作者たちは単純なコピペかそれを機械的に行うツールに頼ることになり、それが常態化した。今はここ。そんなに頭や気を使いたくないのだろうか。

確かにCSSグラデーションの実装と仕様はクソみたいな変遷をたどったが、それはベンダー拡張プリフィックスのせいではない。主に「これコピペするとPhotoshop開かずにグラデーション作れる！すごい！」と考えなしに使ったウェブ制作者だ。

ウェブ制作者たちはブックマークで話題になるウェブログの記事だけではなく、もっとMozilla Developer NetworkやSafari Developer Library、Internet Explorer Dev Center、そしてW3Cの仕様群を読むべきだろう。そしてきちんと実装し、メンテナンスしていくべきだ。

僕は頭や気を使わずに機能を実装しようと思う方が間違っていると思うし、ただでさえコピペでGo！みたいな風潮の強いフロントエンド界隈では、多少頭や気を使って書かなくてはならないようになっていた方が良いと思っている。もちろんその頭や気の使い方というのは、常に手作業で苦労して慎重にタイピングしろというのではない。機能の実装に適切なツールを選定し、ちゃんと理解してそれを使うという程度の話だ。

ベンダー拡張プリフィックスはその機能だけでもウェブ制作者のためになるものだが、それだけではなくウェブ制作者の在り方にもひとつの指針を与えてくれるのではないか。というようなことを考えながら日々ベンダー拡張プリフィックスと戯れている。
