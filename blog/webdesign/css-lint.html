CSS Lint

CSS Lintをしばらく使っていますが、さすがにLintと名乗るだけあって手厳しいですね。効率的でキレイなCSSを書くために自然とHTMLの見直しをすることになる感じが、過渡期の今にはなかなか良いツールな気がします。W3C CSS 検証サービスのCSS3対応がいまいち（border-radius: 4px / 8px;が正しい演算子ではありませんとか怒られたりとか）なこともありますし、もっとみんな使うと良いと思います！ストレスたまる気がしますけどね！

このサイトのCSSだと、

  • セレクタでID使うな
  • widthやheightとpaddingやborderの併用はするな
  • !importantは使うな

この辺りで引っかかります。「承知でやってるよ！」とか言いたくなりますけど、そこはぐっと我慢。

気になるのはDisallow qualified headingsという、見出し（h1-h6）要素はセレクタの先頭以外に使うなというルールです。HTML5だとそれは無理な話な気が……。それともクラス振れとでも言うのでしょうか？

またちょっとCSSいじった
