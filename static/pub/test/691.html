Font Fallback Bug on Android Chrome

AndroidのChrome 80.0.3987.162で、font-familyプロパティーの値が、インストールされていないフォントと総称フォント・ファミリーだけで構成される場合、総称フォント・ファミリーでフォールバックされないことがある。Edge 81、Chrome 81、Firefox 75では適切にフォールバックされる。

Test monospace, sans-serif, and serif

This paragraph should be displayed with “monospace” font.

#test .monospace {
	font-family: "You Do Not Have This Font", monospace;
}

This paragraph should be displayed with “sans-serif” font.

#test .sans-serif {
	font-family: "You Do Not Have This Font", sans-serif;
}

This paragraph should be displayed with “serif” font.

#test .serif {
	font-family: "You Do Not Have This Font", serif;
}

sans-serifはフォールバックされているようだが、monospaceに対してはserif、serifに対してはsans-serifがフォールバックされている。例えばNormalize.cssのfont-family: monospace, monospaceでもserifになってしまう。
